Guia SQL — PostgreSQL

Aprenda a explorar qualquer banco de dados do zero até análise real

1. Quais tabelas existem?

Primeiro passo: descobrir o que existe no banco.


    SELECT table_name
    FROM information_schema.tables
    WHERE table_schema = 'public'
    AND table_type = 'BASE TABLE'
    ORDER BY table_name;
information_schema é o catálogo interno do PostgreSQL — ele mostra toda a estrutura do banco.

2. Quais colunas existem na tabela?

Agora escolha uma tabela e entenda sua estrutura. 

    SELECT
      ordinal_position AS ordem,
      column_name AS coluna,
      data_type AS tipo,
      character_maximum_length AS tamanho,
      is_nullable AS aceita_nulo,
      column_default AS valor_padrao
    FROM information_schema.columns
    WHERE table_name = 'servidor'
    ORDER BY ordinal_position;
USER-DEFINED geralmente significa ENUM (tipo personalizado).

3. Como são os dados?

    -- Ver dados
    SELECT * FROM servidor LIMIT 5;
    
    -- Contagem
    SELECT COUNT(*) FROM servidor;
    
    -- Valores únicos
    SELECT DISTINCT situacao FROM servidor;
Sempre use LIMIT no início. Em produção, tabelas podem ter milhões de registros.

4. Como as tabelas se conectam?


SELECT
  tc.table_name,
  kcu.column_name,
  ccu.table_name AS ref_table,
  ccu.column_name AS ref_column
FROM information_schema.table_constraints tc
JOIN information_schema.key_column_usage kcu
ON tc.constraint_name = kcu.constraint_name
JOIN information_schema.constraint_column_usage ccu
ON ccu.constraint_name = tc.constraint_name
WHERE tc.constraint_type = 'FOREIGN KEY';
Isso mostra exatamente como fazer JOIN entre tabelas.

5. Estatísticas rápidas


SELECT
  COUNT(*) AS total,
  COUNT(salario_atual) AS preenchidos,
  COUNT(*) - COUNT(salario_atual) AS nulos,
  MIN(salario_atual),
  MAX(salario_atual),
  ROUND(AVG(salario_atual),2)
FROM servidor;

SELECT situacao, COUNT(*)
FROM servidor
GROUP BY situacao
ORDER BY COUNT(*) DESC;

6. ENUMs, Views e funções

-- ENUMs
SELECT t.typname, e.enumlabel
FROM pg_type t
JOIN pg_enum e ON e.enumtypid = t.oid;
-- Views
SELECT table_name
FROM information_schema.views;
-- Código da view
SELECT view_definition
FROM information_schema.views
WHERE table_name = 'vw_servidores';
Views já possuem JOINs prontos — use elas como ponto de partida.
